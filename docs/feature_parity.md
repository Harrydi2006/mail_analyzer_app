# 网页端功能还原清单（Flutter）

## 1) 网页端功能盘点

### 首页/系统状态
- 系统状态：邮箱、AI、Notion 连通状态
- 核心统计：邮件总量、事件总量、重要事件、待提醒
- 最近邮件、近期日程

### 邮件管理
- 列表分页、筛选（重要性/处理状态）、关键词搜索
- 流式抓取邮件（SSE 日志、终止任务、恢复展示）
- 单邮件详情：AI总结、提取事件、附件/图片、原文
- 操作：重新分析、失败重试分析、Notion 归档
- 批量操作：选中同步 Notion、整页同步

### 日程表
- 筛选（天数、重要性、关键词）
- 多视图（列表/时间轴/日历）
- 事件详情（含来源邮件跳转）
- 事件操作：编辑重要性、删除、批量删除
- 导出与订阅：iCal 导出、订阅链接、CalDAV 信息

### 系统设置
- 邮箱配置（测试连接）
- AI配置（provider/model/base_url/key + 测试）
- Notion配置（测试、建库、重同步）
- 提醒配置（重要/普通/不重要多档提醒）
- 通知配置（浏览器、Server酱、邮件）
- 关键词管理（重要/普通/不重要）
- 标签管理（订阅规则、历史候选、手动历史标签、重应用）
- 去重 Beta 配置（权重 + 阈值）
- 系统运维（删除全部邮件、系统统计、服务状态）

## 2) Flutter 当前状态

### 已有
- 登录、会话维持
- 首页（状态、统计、最近邮件、近期日程）
- 邮件列表（基础）
- 日程列表（基础）
- 设置页基础项 + 系统操作 + 任务进度

### 本轮已补
- 邮件：筛选（重要性/状态/搜索）、分页、详情弹层
- 邮件：重分析 / 重试分析 / 归档Notion
- 日程：筛选（天数/重要性/搜索）、详情弹层
- 日程：更新重要性、删除
- 设置：关键词管理（重要/普通/不重要）
- 设置：去重 Beta（启用、时间窗口、阈值、权重）
- 设置：标签订阅与历史候选（订阅/取消订阅、手工添加、删除、重应用）
- 邮件：流式处理控制面板（开始/终止/状态/日志）
- 日程：iCal 导出链接与订阅链接（按时间范围和重要性筛选）
- 邮件：图片预览与附件/图片链接复制
- 日程：批量删除（全部或按时间范围）
- 设置：Notion 归档列表与搜索

## 4) 还原策略
- 先以“接口一致 + 交互可用”为目标，不追求与网页同布局
- 每次迭代先补仓库层（repository）再补页面交互
- 每完成一批功能都对照本清单更新状态，保证不漏项
